Правила локализации

Базовый язык

* Базовый язык приложения — английский.
* Localizable.xcstrings сейчас содержит только локаль en.
* Новые языки добавляются как дополнительные локали существующего каталога строк.
* Добавление новой локали не должно требовать изменений бизнес-логики, моделей данных или слоя хранения.
* При отсутствии перевода приложение использует английское значение как резервное.

Источник и поток текстов

* Localizable.xcstrings — основной источник пользовательских строк приложения.
* Системные описания разрешений и другие пользовательские строки из конфигурации приложения также должны иметь локализованные версии.
* Готовые пользовательские тексты нельзя размещать непосредственно в View, ViewModel, Manager, Service, Model или Domain.
* Поток формирования пользовательского текста должен быть следующим:

Domain / Manager
→ смысловое состояние или ошибка
→ presentation mapping
→ View / Toast / Alert / Live Activity

* Domain и Manager возвращают смысловые состояния, результаты и ошибки, но не готовые пользовательские предложения.
* Presentation-слой преобразует состояния и ошибки в локализованный текст.
* Техническая ошибка не должна отображаться пользователю напрямую через localizedDescription.
* ToastManager отвечает только за показ, повтор сообщений и время их отображения, но не за формулировки.

Форматы и данные

* Для строк с параметрами используются format-ключи.
* Для строк, зависящих от количества, используются plural-ключи с правилами конкретного языка.
* Локализованные предложения не собираются конкатенацией отдельных частей.
* Нельзя создавать отдельные варианты готового предложения вручную для разных числительных.
* Порядок параметров не должен зависеть от английской структуры предложения.

Не переводятся автоматически:

* названия треков, артистов, альбомов, жанров, лейблов и другие метаданные музыкальных файлов;
* имена файлов;
* пользовательские названия папок и треклистов;
* реальные файловые пути;
* технические идентификаторы;
* диагностические сообщения, логи и тексты для разработчика.

Внутренние экспортные папки Плеер и Треки сохраняют существующие физические имена ради файлового контракта. Пользовательские названия этих разделов формируются отдельно через presentation-слой.

Добавление новой локали

1. Добавить локаль в Localizable.xcstrings.
2. Перевести все существующие пользовательские ключи.
3. Добавить локализацию системных строк разрешений и конфигурации приложения.
4. Проверить format- и plural-ключи с реальными значениями.
5. Проверить отсутствие непереведённых пользовательских строк и прямого показа технических ошибок.
6. Проверить интерфейс на переносы, обрезание, переполнение и изменение высоты элементов.
7. Проверить View, Toast, sheet, alert, progress, системные уведомления и Live Activity.
8. Проверить доступность: VoiceOver, accessibility labels и accessibility hints.
9. Проверить выбор языка системой и переключение языка приложения, если оно поддерживается iOS.
10. Выполнить ручной проход основных пользовательских сценариев.

Для языков с направлением письма справа налево дополнительно необходимо проверить:

* направление интерфейса;
* порядок иконок и элементов управления;
* выравнивание текста;
* навигационные переходы и жесты;
* отображение пользовательских данных, которые могут оставаться слева направо.
