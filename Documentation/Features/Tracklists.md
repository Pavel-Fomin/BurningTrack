# Назначение

Tracklists — feature-подсистема пользовательских треклистов BurningTrack.

Она позволяет пользователю создавать именованные списки треков, открывать их, менять состав и порядок, запускать действия над треками и использовать треклист как пользовательский набор для воспроизведения или экспорта.

# Пользовательский сценарий

Пользователь открывает Tracklists, чтобы работать со своими треклистами.

На верхнем уровне пользователь видит список треклистов, может создать новый треклист, открыть существующий или удалить ненужный.

Внутри одного треклиста пользователь просматривает треки, меняет их порядок, удаляет треки из треклиста, добавляет новые треки, запускает воспроизведение, открывает подробную информацию о треке, переименовывает трек, переходит к треку в Library, перемещает трек в другую папку или экспортирует треклист.

Если действие требует временного пользовательского сценария, Tracklists использует соответствующий presentation-контракт. Если действие завершается успешно или с ошибкой, Tracklists использует контракт пользовательской обратной связи для результата.

# Основной поток

```text
Пользователь открывает Tracklists
↓
Tracklists показывает пользовательские треклисты
↓
Пользователь создаёт новый треклист или открывает существующий
↓
Пользователь меняет состав, порядок или имя треклиста
↓
Пользователь запускает действие над треком или всем треклистом
↓
Tracklists инициирует соответствующий пользовательский сценарий
↓
Результат возвращается через соответствующий архитектурный контракт.
```

# Участвующие подсистемы

- Root Screens — Tracklists использует Root Screens для подключения как самостоятельной feature-подсистемы и применения общего screen-flow контракта.
- Navigation — Tracklists использует Navigation для перехода от списка треклистов к одному треклисту и связанных пользовательских переходов.
- Runtime Metadata — Tracklists использует Runtime Metadata для отображения актуальных данных треков.
- Track Identity — Tracklists использует Track Identity для действий, относящихся к физическому треку.
- Sheets — Tracklists использует Sheets для временных пользовательских сценариев.
- Errors and Toasts — Tracklists использует Errors and Toasts для пользовательской обратной связи.
- Mini Player — Tracklists использует Mini Player для общей playback presentation внутри раздела Tracklists.
- Library — Tracklists использует Library для перехода к треку в фонотеке.
- Player — Tracklists использует Player для сценариев воспроизведения треков из треклиста.
- Export — Tracklists использует Export для экспорта треклиста.
- Track Edit Session — Tracklists использует Track Edit Session для временных сценариев изменения данных трека.

# Контракты

- Tracklists отвечает за пользовательский сценарий создания, просмотра, изменения и удаления пользовательских треклистов.
- Tracklists отображает готовое состояние треклиста и пользовательские действия над ним.
- Tracklists инициирует создание, переименование и удаление треклистов.
- Tracklists инициирует добавление треков в треклист.
- Tracklists инициирует изменение порядка и удаление треков внутри треклиста.
- Tracklists инициирует воспроизведение треков из треклиста.
- Tracklists использует Track Identity для действий, относящихся к физическому треку.
- Tracklists использует Runtime Metadata для актуальных данных трека.
- Tracklists инициирует временные сценарии через Sheets.
- Tracklists инициирует экспорт треклиста через Export.
- Tracklists использует контракт пользовательской обратной связи для результатов пользовательских действий.

# Ограничения

- Tracklists не описывает внутреннюю реализацию хранения треклистов.
- Tracklists не работает напрямую с источниками данных треков.
- Tracklists не изменяет архитектурные контракты других подсистем.
- Tracklists не превращает временные пользовательские сценарии в часть своего основного feature-flow.
- Tracklists не определяет способ представления пользовательской обратной связи.
