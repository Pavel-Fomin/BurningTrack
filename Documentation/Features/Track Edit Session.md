# Назначение

Track Edit Session — feature-подсистема временного пользовательского сценария изменения данных одного трека.

Она позволяет пользователю открыть данные трека, перейти к редактированию, накопить изменения, подтвердить их одним действием или отменить без сохранения.

# Пользовательский сценарий

Пользователь открывает Track Edit Session из сценария, где доступно действие над конкретным треком.

Сначала пользователь просматривает данные трека. Затем он может перейти к редактированию, изменить данные и выбрать один из двух исходов: подтвердить изменения или отменить сценарий.

Если пользователь подтверждает изменения, Track Edit Session завершает сценарий с результатом. Если пользователь отменяет изменения, сценарий закрывается без применения пользовательского ввода.

# Основной поток

```text
Пользователь открывает Track Edit Session
↓
Track Edit Session отображает данные выбранного трека
↓
Пользователь переходит к редактированию
↓
Track Edit Session фиксирует исходное состояние сценария
↓
Пользователь изменяет данные трека
↓
Пользователь подтверждает или отменяет изменения
↓
Track Edit Session завершает пользовательский сценарий
↓
Результат возвращается через соответствующий архитектурный контракт.
```

# Участвующие подсистемы

- Sheets — Track Edit Session использует Sheets для временного пользовательского сценария поверх текущего feature-flow.
- Runtime Metadata — Track Edit Session использует Runtime Metadata для актуальных данных трека и обновления отображаемого состояния после изменения.
- Track Identity — Track Edit Session использует Track Identity для действий, относящихся к физическому треку.
- Errors and Toasts — Track Edit Session использует Errors and Toasts для пользовательской обратной связи по результату сценария.

# Контракты

- Track Edit Session отвечает за пользовательский сценарий изменения данных одного трека.
- Track Edit Session отображает данные трека до входа в редактирование.
- Track Edit Session отображает временное состояние редактирования до завершения сценария.
- Track Edit Session получает исходные данные через Runtime Metadata.
- Track Edit Session получает пользовательские изменения как временное состояние сценария.
- Track Edit Session использует Track Identity для действий, относящихся к физическому треку.
- Track Edit Session инициирует применение только подтверждённых изменений.
- Track Edit Session передаёт отмену как завершение сценария без применения пользовательского ввода.
- Track Edit Session передаёт результат изменения через Runtime Metadata.
- Track Edit Session передаёт успешный результат или ошибку в соответствующий feedback-контракт.

# Ограничения

- Track Edit Session не описывает внутреннее выполнение изменения данных трека.
- Track Edit Session не применяет пользовательский ввод без подтверждения.
- Track Edit Session не сохраняет частичные изменения после отмены.
- Track Edit Session не задаёт правила открытия временных сценариев.
- Track Edit Session не определяет способ представления пользовательской обратной связи.
- Track Edit Session не описывает массовое редактирование выбранного набора треков.
- Track Edit Session не определяет архитектурные правила других feature-подсистем.
