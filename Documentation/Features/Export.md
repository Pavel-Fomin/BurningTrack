# Назначение

Export — feature-подсистема передачи выбранного набора треков за пределы BurningTrack.

Она позволяет пользователю запустить экспорт из сценария, где уже сформирован набор треков, и получить результат выполнения через общий контракт пользовательской обратной связи.

# Пользовательский сценарий

Пользователь запускает Export из feature-сценария, где доступен экспорт набора треков.

Если выбранный набор можно экспортировать, Export запускает пользовательский сценарий передачи треков за пределы приложения.

Если экспорт невозможен или завершается с ошибкой, пользователь получает результат через существующий feedback-контракт.

# Основной поток

```text
Пользователь запускает Export
↓
Export получает выбранный набор треков
↓
Export проверяет, что набор можно экспортировать
↓
Export инициирует передачу выбранного набора за пределы приложения
↓
Export завершает пользовательский сценарий
↓
Результат возвращается через соответствующий архитектурный контракт.
```

# Участвующие подсистемы

- Player — Export использует Player для сценария экспорта текущего списка воспроизведения.
- Tracklists — Export использует Tracklists для сценария экспорта пользовательского треклиста.
- Track Identity — Export использует Track Identity для действий, относящихся к физическим трекам.
- Errors and Toasts — Export использует Errors and Toasts для пользовательской обратной связи по результату экспорта.

# Контракты

- Export отвечает за пользовательский сценарий передачи выбранного набора треков за пределы приложения.
- Export отвечает за отделение экспортного сценария от playback-сценариев и сценариев пользовательских треклистов.
- Export получает выбранный набор треков из feature-сценария, который инициировал экспорт.
- Export использует Track Identity для действий, относящихся к физическим трекам.
- Export инициирует передачу выбранного набора треков за пределы приложения.
- Export передаёт ошибку или успешный результат в соответствующий feedback-контракт.

# Ограничения

- Export не описывает способ подготовки и передачи треков за пределы приложения.
- Export не изменяет состав текущего списка воспроизведения.
- Export не изменяет состав пользовательского треклиста.
- Export не определяет playback-состояние.
- Export не выполняет пользовательский сценарий без выбранного набора треков.
- Export не определяет способ представления пользовательской обратной связи.
- Export не определяет архитектурные правила других feature-подсистем.
