# Назначение

Batch Operations — feature-подсистема массовых пользовательских действий над выбранным набором треков.

Она позволяет пользователю выбрать несколько треков, запустить действие над этим набором, уточнить параметры или подтвердить выполнение и получить итоговый результат.

# Пользовательский сценарий

Пользователь выбирает несколько треков в сценарии, где доступен режим массового выбора.

После выбора пользователь запускает массовое действие. Если действию нужны дополнительные параметры или подтверждение, пользователь проходит временный сценарий подготовки.

После выполнения пользователь получает результат всего массового действия: успешное завершение, частичный результат или ошибку.

# Основной поток

```text
Пользователь выбирает несколько треков
↓
Пользователь выбирает массовое действие
↓
Batch Operations фиксирует выбранный набор
↓
Пользователь уточняет параметры или подтверждает выполнение, если это требуется
↓
Batch Operations инициирует выполнение массового сценария
↓
Batch Operations завершает пользовательский сценарий
↓
Результат возвращается через соответствующий архитектурный контракт.
```

# Участвующие подсистемы

- Library — Batch Operations использует Library для запуска массового сценария от выбранного пользовательского набора треков.
- Runtime Metadata — Batch Operations использует Runtime Metadata для готовых данных выбранных треков.
- Track Identity — Batch Operations использует Track Identity для действий, относящихся к физическим трекам.
- Sheets — Batch Operations использует Sheets для временных сценариев ввода, подготовки или подтверждения.
- Errors and Toasts — Batch Operations использует Errors and Toasts для итоговой пользовательской обратной связи.

# Контракты

- Batch Operations отвечает за пользовательский сценарий действия над несколькими выбранными треками.
- Batch Operations отвечает за общий пользовательский сценарий массового действия независимо от конкретного типа массового действия.
- Batch Operations использует упорядоченный выбранный набор как вход пользовательского сценария.
- Batch Operations отображает готовое состояние подготовки, подтверждения или выполнения массового действия.
- Batch Operations инициирует выполнение выбранного массового сценария после пользовательского подтверждения, если подтверждение требуется.
- Batch Operations использует соответствующий архитектурный контракт для действий, относящихся к физическим трекам.
- Batch Operations использует соответствующий архитектурный контракт для актуальных данных выбранных треков.
- Batch Operations инициирует временные сценарии через Sheets, если массовому действию нужен ввод пользователя.
- Batch Operations инициирует итоговую пользовательскую обратную связь после завершения массового сценария.
- Batch Operations отображает результат массового действия как итог всего выбранного набора, включая частичное выполнение, если оно возможно для сценария.

# Ограничения

- Batch Operations не описывает внутреннюю реализацию выполнения массовых действий.
- Batch Operations не подменяет одиночные действия над треком.
- Batch Operations не вводит закрытый список массовых действий как архитектурный контракт.
- Batch Operations не выполняет пользовательское действие без выбранного набора треков.
- Batch Operations не хранит пользовательскую фонотеку или данные треков как постоянный источник правды.
- Batch Operations не превращает временные сценарии подготовки или подтверждения в свой постоянный feature-flow.
- Batch Operations не определяет способ представления пользовательской обратной связи.
